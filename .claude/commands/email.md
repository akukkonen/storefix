Write an email template for AccessFix.

Type: $ARGUMENTS

If no type specified, list the available types and ask.

## Email types

- **waitlist-confirm** — Auto-reply when someone joins the waitlist. Short, warm, sets expectations for launch timeline.
- **launch** — Announcement to waitlist that AccessFix is live. Key features, free scan CTA, install link.
- **onboarding** — Welcome after install. Guide to first scan in 3 steps max.
- **upgrade** — Pitch to free-tier users with >3 issues. Reference their scan results, explain Fix plan value.
- **follow-up** — Check-in 7 days after install for users who haven't scanned. Gentle, not pushy.
- **custom** — Anything else described in the arguments.

## Guidelines

- **From:** AccessFix <support@getaccessfix.app>
- **Tone:** Friendly expert. Not corporate ("Dear valued customer"), not fake-casual ("Hey!!! 🎉"). Write like a helpful colleague.
- **Length:** Under 150 words. Merchants don't read long emails.
- **Structure:** Subject line → 1-2 short paragraphs → single CTA button/link → sign-off
- **CTA:** One clear action. Never give 3 options.
- **Footer:** Include "Unsubscribe" note.
- Don't use emoji in subject lines. Don't use ALL CAPS. Don't use "!" more than once.

Output as plain text with Subject line clearly marked.
